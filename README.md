Общие требования к проекту:
Проект должен был развернут на одном из бесплатных хостингов в
сети интернет, например, Firebase Hosting.
Требования к интерфейсу:
Интерфейс приложения должен быть выполнен с использованием
HTML 5. Разметка должна быть семантически корректна, например,
тег <a> испольуется для ссылок, а не кнопок. Различные части
страницы должны использовать существующие HML5 теги для ясной и
четкой структуры приложения для разработчиков (<nav>, <header>,
<article>, ...). Пройти валидацию здесь: https://validator.w3.org/
Для определения стилей используется CSS. Не разрешено
использование дополнительных библиотек, таких, как bootstrap.
Любое включение доп. библиотек должно быть согласовано в
индивидуальном порядке.
Основные усилия нужно уделить разметке и четкой структуре
проекта. Интерфейс приложения должен быть понятен пользователю
с первого взгляда.

Интерактивность приложения
Фронтенд приложения должен быть написан на чистом JS. Не
разрешено использование дополнительных библиотек, за
исключением, Firebase/ Amazon connectivity libraries. Любое
включение библиотек должно быть согласовано в индивидуальном
порядке.
Использование фреймворков
Идея курса заключатеся в изучении чистого JS, как языка для вебразработки. Не рекомендуется использование фреймворков, т.к.
проверки знаний будут основаны на знании JS. Если вы хотите
использовать какой-либо фреймворк, то согласовать это можно до
выбора темы. В конце семестра проекты, использующие фреймворки,
могут быть не приняты.
Что изучить:
• JavaScript: The Definitive Guide — a good place to start
• JavaScript: The Good Parts
• JavaScript: The Right Way
• Codecademy's introduction to JS — alternative place to start
• MDN's JavaScript section
• Clean code concepts: JS version
• Building SPA in plain JS

Критерии оценки.
Разметка
[4-6] знание основных тегов (div, span, img, input, ul, p...).
[7-8] семантически корректная разметка (header, footer, article,
nav...).
[9-10] богатая семантика (figure, picture, aside...), разметка с
соответсвующей иерархией заголовков (h1, h2...).
CSS
Т.к. в этом курсе мы не изучаем дизайн, то здесь нет строгих
требований к оформлению вашего приложения. Однако более-менее
понятный UI обязателен, т.к. это мотивирует использовать большее
количество элементов CSS.
[4-6] базово использование CSS. Кастомизированные формы
ввода, кнопки и другие элементы, :hover и :disabled для интерактивных
элементов, отображение: flex block positioning.
[7-8] отображение: flex or display: grid block позиционирование,
:focus состояние для интерактивных элементов (отличное от :hover),
[9-10] широкое использование анимации и хороший визуальный
стиль
Основы фронтенда
Необходимо разработать рабочий прототип веб-сайта: базовая
навигация, аутентификация и др. Функции
[4-6] базовые JS функции, переменные (использование var
запрещено), циклы, массивы.
[7-8] классы, AJAX запросы (через Fetch API), promises.
[9-10] SPA implementation (via browser's History API), web
components, async / await, локальное хранилище.
